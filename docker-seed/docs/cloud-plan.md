# Cloud access (outline)

## Goal

Make the workspace and dashboard reachable from more places (phone, browser, remote desktop) while keeping a clear source of truth for data and documents.

## How you get there

The exact stack (hosting provider, database service, authentication, and file sync) is **chosen and documented by whoever operates your deployment**, not in this seeded workspace folder.

Use **`docs/system.md`** for how this tenant is laid out, and ask your administrator for URLs, credentials, and any migration or backup runbooks.
